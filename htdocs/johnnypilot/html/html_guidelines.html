HTML/XHTML Guidelines

html_guidelines_v1.0.doc - 26.05.00

 

Introduction

These guidelines have been created for the following reasons:

  1. To encourage use of a "house style" of html code which all developers are familiar with;
  2. To facilitate the current and future use of html code produced now within emerging stylesheet languages such as xsl;
  3. To enable sophisticated global "grep pattern" searching throughout every document of a web site.

 

HTML guidelines

NB Many of these rules are mandatory for XHTML, optional for HTML, but should be followed nonetheless. The rules specific to XHTML only are listed separately - these should be followed in addition to the HTML guidelines when producing HTML template files for incorporation into XSL documents. There is also an html template file (html_guidelines_template.html) containing examples of the relevant code, including document type definitions and suitable meta tags.

  1. All attribute values should be quoted. For example, <font size="2">.
  2. All tags should be lowercase. NB attribute values may be either case, eg <font color="#FF00FF">.
  3. All elements should have a closing element. For example, <p>…</p>
    NB when preparing xhtml for use in xsl stylesheets see additional notes below regarding non-paired tags.
  4. All elements should nest properly. For example, <p><b>…</b></p>.
  5. Non-paired tags such as <br> or <img> should be modified to include a "slash" at the end,
    eg use the single element <br />. Note the white space before the slash.
  6. All alphanumeric characters will have to be expressed as entities.
  7. When an attribute value contains an ampersand, it should be expressed as an entity reference. For example, when the "href" attribute of the <a> element refers to a CGI script that takes parameters, it should be expressed as:
    http://www.canon.co.uk/cgi-bin/parser.pl?page=products&amp;product=630.
  8. Certain element attributes should always be present, and should be stated in specific order.
    See file "html_guidelines_template.html".

 

XHTML guidelines - specific to implementation of XSL stylesheets

1. Attribute minimization is forbidden; every attribute must have a value, eg <td nowrap> must be written as <td nowrap="nowrap">.

2. The id attribute replaces the name attribute of HTML 4.0. This is only relevant to the elements a, applet, frame, iframe, img, and map.

3. Script and Style elements must be enclosed within "CDATA" sections:

<script>
<![CDATA[
... script content ...
]]>
</script>

4. There are some mandatory elements:
i the DOCTYPE declaration
;; the <html> element must include a namespace declaration
iii the <head> element must be present and include a title element
iv the <body> element must be present.

 

Meta Tags

To enhance accessibility to search engines which use web spiders to index pages always specify keywords and a page description within the meta tags. Try to use "pairs" of keywords, and also incorporate as many keywords in the page’s <title> tags as possible. Although it’s generally the author’s responsibility, the recommendation to clients should be that keywords should appear as early as possible within the <body> tags as well. We can help here by choosing keywords selectively from the first few paragraphs.

 

Supplementary attributes for non-visual browsers

Certain attributes should be added to enhance accessibility to information by non-graphical and aural browsers.

1. "alt" attribute should always be used to give a short description of images within the <img> element. If necessary a "longdesc" attribute may be used to reference an external descriptive file.

2. "summary" attribute may be added to <table> element to give a description of the table’s purpose.

3. <label> element may be used to tag form elements with a description of their purpose, eg:

<form action="mailto:admin@realtime.co.uk">

<label>Name<br><input name="Name" size=30></label><br />

<label>Address<br><input name="Address" size=45></label><br />
